# Claude Code Setup

> AI Agent configuration — copy to any project to get standardized Claude workflows.

## Quick Start

```bash
cp -r claude-setup/. your-project/
chmod +x .claude/hooks/*.sh
cd your-project
claude
```

### Step 2: Tell Claude your project

```
I want to set up my project. Here's my project info:
- Name: YOUR_PROJECT_NAME
- Type: web app / api / mobile app / smart contracts / etc
- Tech stack: React + Node.js + PostgreSQL / Next.js + Supabase / etc
- Core feature: WHAT_IT_DOES
```

Claude will fill `CLAUDE.md` with appropriate patterns and placeholders.

### Step 3: Generate docs/ folder

```
Run /checkpoint to generate the docs/ folder templates.
```

Claude auto-creates:

- `docs/spec-doc.md` — Vision, goals, requirements
- `docs/architecture.md` — System design
- `docs/project-plan.md` — Roadmap and milestones
- `docs/project-status.md` — Session tracking
- `docs/changelog.md` — Change history

### Step 4: Fill project docs

Tell Claude to help fill each doc:

```
Please help me fill docs/spec-doc.md with:
- Project overview and goals
- Core features (MVP first)
- User stories
```

### Step 5: Generate roadmap

```
Run /generate-plan to create the implementation roadmap.
```

### Step 6: Create project-specific agents/commands (optional)

Tell Claude:

```
My project needs a [backend-test-agent / deploy command / etc]. Please create it.
```

### Step 7: Build

After each session, run `/checkpoint` to sync all docs.

## Example Conversation

```
You: I want to set up a new project called "Blog CMS" using Next.js + PostgreSQL + Prisma

Claude: Great! Based on what you told me:
- Project: Blog CMS
- Stack: Next.js 14, PostgreSQL, Prisma, TypeScript
- Core: Content management with markdown support, user roles

Let me fill CLAUDE.md with the appropriate patterns and placeholders for you.

Done! CLAUDE.md is ready with:
- Project name and description
- Tech stack patterns (App Router, Prisma service pattern)
- TypeScript strict mode rules
- Environment variables template

Now run `/checkpoint` to generate your docs/ folder.

---

You: /checkpoint

Claude: Created docs/ folder with 5 template files.

Now please tell me about your Blog CMS:
1. Who will use it? (bloggers, companies, internal team)
2. What are MVP features? (create posts, edit, publish, categories)
3. Any special requirements? (markdown, image upload, SEO)

---

You: [answers questions]

Claude: I've filled all the docs:
- spec-doc.md with your vision and MVP features
- architecture.md with your tech stack design
- project-plan.md with milestones
- project-status.md with initial state

Ready to start? Run `/generate-plan` for detailed tasks.
```

## Directory Structure

```
your-project/
├── CLAUDE.md                      # Project brain - Claude reads this first
├── .env.example                   # Environment variables template
│
├── .claude/
│   ├── settings.json               # Permissions and hooks
│   ├── commands/
│   │   ├── commit.md              # Conventional commits
│   │   ├── pr.md                  # Create pull request
│   │   ├── checkpoint.md           # Sync changelog + status + plan
│   │   └── generate-plan.md       # Create implementation plan
│   ├── agents/
│   │   ├── backend-test-agent.md  # Backend unit/integration tests
│   │   ├── frontend-test-agent.md # Playwright UI tests
│   │   └── research-agent.md       # Tech stack research
│   └── hooks/
│       ├── on-stop.sh             # After task completes
│       ├── auto-lint.sh           # After file edits
│       └── pre-bash-check.sh      # Before bash commands
│
└── docs/                          # Auto-generated by /checkpoint
```

## Commands

| Command          | When                                                             |
| ---------------- | ---------------------------------------------------------------- |
| `/checkpoint`    | Sync docs after each session (also generates docs/ on first run) |
| `/commit`        | Create conventional commit                                       |
| `/pr`            | Create pull request                                              |
| `/generate-plan` | Create detailed implementation roadmap                           |

## Docs Hierarchy

```
CLAUDE.md → docs/spec-doc.md → docs/architecture.md → docs/project-plan.md
```

Claude reads these in order at session start. Keep them updated.
